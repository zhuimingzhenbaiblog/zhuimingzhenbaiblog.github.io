c++补充知识3.0

int a 和 int &a 的区别：
int a的意思是定义一个变量a
int &a意思是定义一个引用 //引用相当于指针再取值 他和被引用的变量都是表示同一块内存
引用就是给变量取别名 int b ;int &a=b; 用cout<<&a 和cout<<&b 你会发现他们的地址是一样的

C++核心编程：
内存分区模型：
#代码区：存放函数体的二进制代码，有操作系统进行管理。（就是自己写的代码的内容就是放在代码区中）
#全局区：存放全局变量和静态变量以及常量。
#栈区：由编译器自动分配释放，存放函数的参数值，局部变量等。（该地区的数据存活死亡由编译器管理）
#堆区:由程序员分配和释放，若程序员不释放，程序结束时由操作系统回收。
内存分区的意义：
给数据不同的生命周期啊，内存管理会更加方便，合理，反正大大的好。

关于代码区：
    1.很明显，存放的CPU执行的机器指令。就是01010101的二进制。
    2.代码区是共享的，因为有些程序是频繁执行的，所以共享的话只要在内存中有一份代码就好了。
    3.代码区是只读的，为了防止被改，你懂得。

关于全局区：
    全局变量和静态变量都放在这。
    还包含了常量区:字符串常量和其他常量。
    注意！！ 该区域的数据在程序结束后由操作系统释放，不归程序员管哦。
    #全局变量定义在函数的外面
    #静态变量：在普通变量前面加static后就属于静态变量，存放在全局区
        语法： static int s_a = 10;
    #常量：
        1.字符串常量"Hello World"
        2.const修饰的全局变量（在全局区）
        3.const修饰的局部变量（不在全局区）

关于栈区：
    注意事项：不要返回局部变量的地址，栈区开辟的数据由编译器自动释放
    栈区存放局部变量和形参的数据

关于堆区：
    利用new关键字，可以将数据开辟到堆区
    new int(10); //返回这个数据的指针
    int * p = new int (10);  //接收这个数据，此时指针本质也是局部变量，反正栈上，指针保存的数据是放在堆区
    释放语法： delete p;  //释放p的内存
    new int[10];  //创建一个数组
    int * arr = new int[10];   
    delete[] arr;       //释放数组

引用：
给变量起别名。
语法： 数据类型 &别名 = 原名；
例子: int a =10; int &b = a;
注意事项： 引用必须初始化，并且不可以改。
##引用做函数参数
引用传递 void swap(int &a , int &b){…}

引用做函数的返回值：
    不要返回局部变量的引用。
    函数的调用可以作为左值。 左值： test() = 1000; 

引用的本质：
    引用的本质在c++内部实现是一个指针常量
    int & ref = a; //等效于 int * const ref = &a; 指针的指向不可以改，值可以改

函数的重载：
作用：函数的名可以相同，提高复用性
满足的条件：
1.同一个作用域下
2.函数名称相同
3.函数的参数类型不同，或者个数不同或者顺序不同

类和对象：
三大特性：封装，继承，多态

    ##将属性和行为加以权限控制

数组，结构体，类的区别:
    数组是一系列类型相同的元素集合。
    结构体是不同类型的元素的集合。
    类不仅是数据类型不同的元素组合，还包括了对元素的操作方法。
    struct 默认权限是 公共 public
    class  默认权限是 私有 private

封装的意义：
    类在设计的时候，可以把属性和行为放在不同的权限下，加以控制。
    三种权限：
        1.public     公共权限  类内可以访问 类外可以访问
        2.protected  保护权限  类内可以访问 类外不可以访问
        3.private    私有权限  类内可以访问 类外不可以访问
    成员属性设置为私有：

        优点2.对于写权限，我们可以检测数据的有效性

对象的初始化和清理：

2.显示法

3.隐式转换法

拷贝构造函数的调用时机:
    1.使用一个已经创建完毕的对象来初始化一个新对象
    2.值传递的方式给函数参数传值

空指针：
指向内存中编号为0的空间
用途：初始化指针变量（如一开始不知道指针指向哪好的时候就指向编号为0的空间）
注意：空指针指向的内存是不可以访问的
int * p = NULL;

野指针：
int * p = (int )0x1100; //指针变量p指向内存地址编号为0x1100的空间
cout<<p << endl; //访问野指针报错

结论：空指针和野指针都不是我们申请的空间，所以不要随便的访问他们

const int * p = &a;(常量指针)：
特点：指针的指向可以修改，但是指指向的值不可以改
*p = 20; //错误
p = &b; //正确

int * const p = &a;(指针常量)：
特点：指针的指向不可以改，指针指向的值可以改
*p = 20; //正确
p = &b; //错误

const int * const p = &a;(即修饰指针，又修饰常量)：
特点：指针的指向和指针指向的值都不可以改
*p = 20; //错误
p = &b; //错误

冒泡排序函数：
void bubbleSort(int* arr, int len)
{
for (int i = 0; i < len - 1; i++)
{
for(int j = 0; j < len - i - 1; j++)
if (arr[j] > arr[j + 1])
{
int temp = arr[j];
arr[j] = arr[j + 1];
arr[j + 1] = temp;
}
}
}
打印函数：

和队友一起失败的滋味比单独一人失败要难受的多，你没办法对他们讲出放心这个交给我，没办法使你成为依靠。自己觉得自己是一个不靠谱的人那是有多糟糕啊
然而这次毕业设计级别的课程设计，我不是孤独一人
可以说是续那场竞赛之后，第一个team
三个人 和一个共同实现的目标
我有反问过自己，自己可以成为别人的依靠了吗？靠谱了吗？
答案很显然：并没有
但也许这次可以使我真正的成长了
三个不靠谱的男人的结果会是什么样的呢？
。。。。。。。。。

直面自己的内心，都是他夜里躺着床上睡前才会做的事情。睡一觉后起个床后，负心汉就会完全忘记自己昨晚对自己定下的目标[○･｀Д´･ ○]

但我知道这次的课程设计对于他的重要性，不管是哪个方面的
3个月的时间来揭晓谜底
如果失败了会是怎样的一番情况？
如果成功了又会是怎么的呢？
